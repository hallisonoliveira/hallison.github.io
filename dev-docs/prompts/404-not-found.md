# Contexto

Hoje, é possível ter posts em apenas um dos idiomas suportados. Quando isso acontece, ao mudar de idioma estando na página de um post que não possui tradução, o usuário é redirecionado para uma página de erro 404. Esse comportamento é esperado, mas faz-se necessário ter uma página customizada para casos de 404 gerais no blog.

# Requisitos
- Criar uma página "Página não encotrada" para ser exibida em casos de 404
- Mostrar essa página customizada em qualquer caso de 404
- A página deve seguir o design já estabelecido no blog
- A página deve ser exibida em ingles ou portugues, de acordo com o idioma selecionado
